Data Layer

Database Architecture

Store Technology What Lives Here Scaling Strategy
Primary DB PostgreSQL (Neon) Users, orders, products, addresses, payments Serverless autoscale, read replicas, connection pooling

Request Lifecycle

End-to-End Data Flow

1
User Visits Product Page
Browser hits flawlessera.store/products/[slug]
→ Cloudflare edge receives request
→ Check CDN cache (static HTML)
→ Cache HIT → serve in < 50ms
→ Cache MISS → Next.js SSR
2
Server Side Render (SSR)
Next.js server component fetches product data
→ Check Redis cache first
→ Cache MISS → query PostgreSQL
→ Drizzle ORM typed query
→ Store in Redis (TTL 5 min)
→ Return full HTML to browser
3
User Searches Products
Instant search as user types
→ Typesense InstantSearch UI
→ Query hits Typesense directly
→ Results in < 50ms with filters
→ No PostgreSQL hit needed
4
Add to Cart
Anon or logged-in cart management
→ Zustand updates local state instantly
→ API call → Order Service
→ Cart persisted in Redis (session)
→ Anon cart → merge on login
5
Checkout + OTP Login
Anonymous phone-OTP flow (no forced signup)
→ Enter phone → MSG91 sends OTP
→ Verify OTP → Better Auth session
→ Silent user creation in PostgreSQL
→ Cart merged → checkout continues
6
Payment Processing
Razorpay integration flow
→ Create Razorpay order (backend)
→ Open Razorpay checkout UI
→ User pays (UPI/Card/COD)
→ Razorpay webhook → verify sig
→ Order status updated in PG
7
Post-Payment Notifications
Multi-channel order confirmation
→ BullMQ job triggered
→ Resend → email confirmation
→ MSG91 → SMS confirmation
→ WhatsApp API → order details
→ Shiprocket → shipment created
8
Admin Dashboard
admin.flawlessera.store
→ tRPC typed API calls
→ RBAC — admin/vendor roles
→ Product CRUD, inventory
→ Order management, exports
→ Analytics via PostHog embed
